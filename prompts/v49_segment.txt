Rules:

2. Use the cycle pattern as a guide only. Visible off-cycle motions such as table/pile management, clearing, recovery, failed attempts, dropped items, or tool/object repositioning get their own cuts when they form complete visible sub-goals.
3. A valid segment contains one complete, labelable visible sub-goal or task phase — one ATOMIC action (a single verb on a single object-state change), NEVER "a whole task got completed". When the hand switches from one contacted tool/object to a different tool/object or to bare fingers, that is a purpose change — cut there. 4. Do not cut inside continuous motion on the same object toward the same goal, even if it crosses clock seconds. Regrips, small corrections, repeated tries without completed release, and continuous handling of a handful/group stay in one segment.

If sampled "release" moments look like the "pick/handle" moments, collapse into one continuous-action segment (gather/tuck/arrange/feed/pack).
5. Phase ontology for source->work->destination cycles: source acquisition / transport / active manipulation / post-placement adjustment / release-deposit. Cut between phases only when their visible goals differ; acquisition is not insertion, pressing is not picking, and transport is not target work.
6. Multi-cycle anti-swallowing guard: audit every proposed segment by sampling start, 25%, 50%, 75% and end (for spans over 12s, also every ~4-6s). A segment is INVALID if it contains more than one completed cycle of acquire -> work -> finish -> destination move/release -> next acquire. Split triggers: a completed release/deposit followed by regrasping another item; hands leave the work area for a source/destination and return with a new item; one finished part deposited then a different part acquired; separation/joining completes then the next acquisition begins. Never let one segment bridge deposit->regrasp->new manipulation merely because the same verb repeats; if unsure whether a release completed, request a burst instead of swallowing the next cycle. Also split when a segment alternates distinct long work phases (>~20s each).
7. Separation->destination cycles: do not start the drop/sort/place segment while hands are still together at the central work area twisting, pinching, or finishing separation. Start it at the first visible transition to destination-directed transport, when separated item(s) move outward/downward toward container openings.
8. The destination-transfer/drop segment is the destination-directed transport plus release. Never pad it backward into manipulation to make it longer.
8b. Release visibility requirement: never infer a release from small hand motion, rhythmic repetition or expected task structure — if release completion is not visible, keep it inside the continuous span or request a burst.

10. Routine contact-state changes are not standalone segments: final release of a completed object belongs with the preceding action, and initial reach/grasp of the next object belongs with the following action.
