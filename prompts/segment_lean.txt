Rules:

2. Visible off-pattern motions such as table/pile management, clearing, recovery, failed attempts, dropped items, or tool/object repositioning get their own cuts when they form complete visible sub-goals.
3. A valid segment contains one complete, labelable visible sub-goal — one ATOMIC action (a single verb on a single object-state change), NEVER "a whole task got completed". When a hand switches from one contacted object to a different object or to bare fingers, that is a purpose change — cut there. Cut only when the visible purpose changes: new object, new destination, new task phase, or fundamentally different hand/object motion.
4. Do not cut inside continuous motion on the same object toward the same goal, even if it crosses clock seconds. Regrips, small corrections, repeated tries without completed release, and continuous handling of a handful/group stay in one segment.
4b. Continuous handling vs discrete micro-cycles: do NOT create repeated pick/release cycles for continuous handling of the same handful, pile, cluster, flexible group or container mouth when the hand stays in contact or immediately recontacts without a visible completed let-go. A discrete release segment requires VISIBLE completion (object no longer hand-controlled; the releasing hand open/empty or clearly separated). If sampled "release" moments look like the "pick/handle" moments, collapse into one continuous-action segment.
5. Phase ontology for source->work->destination cycles: source acquisition / transport / active manipulation / post-placement adjustment / release-deposit. Cut between phases only when their visible goals differ; acquisition is not insertion, pressing is not picking, transport is not target work.
6. Multi-cycle anti-swallowing guard: audit every proposed segment by sampling start, 25%, 50%, 75% and end (for spans over 12s, also every ~4-6s). A segment is INVALID if it contains more than one completed cycle of acquire -> work -> finish -> destination move/release -> next acquire. Split triggers: a completed release/deposit followed by regrasping another item; hands leave the work area and return with a new item; one finished part deposited then a different part acquired; separation/joining completes then the next acquisition begins. Never let one segment bridge deposit->regrasp->new manipulation merely because the same verb repeats.
7. Separation->destination cycles: do not start the drop/sort/place segment while hands are still together at the central work area finishing separation. Start it at the first visible transition to destination-directed transport.
8. The destination-transfer/drop segment is the destination-directed transport plus release. Never pad it backward into manipulation to make it longer.
8b. Release visibility: never infer a release from small hand motion, rhythmic repetition or expected task structure — if release completion is not visible, keep it inside the continuous span.
9. A release/drop is a separate segment only when it is the visible goal of a destination-transfer/drop phase or is detached from carrying/placing by a pause, independent let-go, separate fall, slip, impact, or recovery.
10. Routine contact-state changes are not standalone segments: final release of a completed object belongs with the preceding action, and initial reach/grasp of the next object belongs with the following action.
11. In an end-of-cycle transition, use at most ONE boundary between the completed-object phase and the next raw-part acquisition phase; no separate sub-second release and grasp fragments.
12. Be conservative. Typical tabletop sub-actions are about 2-8s.
13. Protected short exception: in repetitive sorting/binning tasks, a destination-transfer/drop phase may be 0.4-1.5s when it visibly contains movement toward a destination container and/or release into it.
